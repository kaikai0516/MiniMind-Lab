```bash

```

## 三、NVIDIA GPU 部署

### 3.1 前置条件

```bash
# 确认驱动和 CUDA 版本
nvidia-smi
# 应输出: NVIDIA-SMI 5xx.xx  Driver Version: 5xx.xx  CUDA Version: 12.x
```

### 3.2 安装 PyTorch

根据你的 CUDA 版本选择：

```bash
# CUDA 12.6 / 12.4 (最新)
pip install torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124

# CUDA 12.1
pip install torch==2.5.1 torchvision==0.20.1 --index-url https://download.pytorch.org/whl/cu121

# CUDA 11.8 (旧卡如 GTX 10 系列)
pip install torch==2.3.0 torchvision==0.18.0 --index-url https://download.pytorch.org/whl/cu118
```

### 3.3 验证

```bash
python -c "
import torch
print(f'CUDA available: {torch.cuda.is_available()}')
print(f'GPU name: {torch.cuda.get_device_name(0)}')
print(f'GPU memory: {torch.cuda.get_device_properties(0).total_memory / 1e9:.1f} GB')
print(f'CUDA version: {torch.version.cuda}')
"
# 期望输出:
# CUDA available: True
# GPU name: NVIDIA GeForce RTX 3090
# GPU memory: 24.0 GB
# CUDA version: 12.4
```

### 3.4 常见 NVIDIA 问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `torch.cuda.is_available() = False` | CUDA 版本不匹配 | 检查 `nvidia-smi` 显示的 CUDA 版本 |
| `CUDA out of memory` | 显存不足 | 减小 `--batch_size`, 降低 `--hidden_size` |
| `cuDNN error` | cuDNN 版本兼容 | 升级 PyTorch 版本 |
| DLL 冲突 (Windows) | pyarrow 和 torch 冲突 | `import datasets` 在 `import torch` 之前 |

## 四、AMD GPU 部署 (ROCm)

### 4.1 前置条件

```bash
# 确认 ROCm 安装
rocm-smi
# 应输出 GPU 信息和 ROCm 版本号

# 确认系统内核
uname -m && cat /etc/os-release
# ROCm 6.x 需要 Ubuntu 22.04 / RHEL 9+
```

### 4.2 安装 PyTorch (ROCm)

```bash
# ROCm 6.2 (最新, 推荐)
pip install torch==2.6.0 torchvision==0.21.0 \
    --index-url https://download.pytorch.org/whl/rocm6.2

# ROCm 6.1
pip install torch==2.5.1 torchvision==0.20.1 \
    --index-url https://download.pytorch.org/whl/rocm6.1

# ROCm 5.7 (旧版)
pip install torch==2.3.0 torchvision==0.18.0 \
    --index-url https://download.pytorch.org/whl/rocm5.7
```

### 4.3 验证

```bash
python -c "
import torch
print(f'ROCm available: {torch.cuda.is_available()}')
print(f'GPU name: {torch.cuda.get_device_name(0)}')
print(f'HIP version: {torch.version.hip}')
print(f'GPU memory: {torch.cuda.get_device_properties(0).total_memory / 1e9:.1f} GB')
"
# 期望输出:
# ROCm available: True
# GPU name: AMD Radeon RX 7900 XTX
# HIP version: 6.2.x
# GPU memory: 24.0 GB
```

### 4.4 AMD GPU 兼容性

MiniMind-Lab 已完成 ROCm 适配：

| 功能 | 状态 | 说明 |
|------|------|------|
| `torch.cuda.*` API | ✅ 正常 | ROCm 提供完整 CUDA API 兼容层 |
| bfloat16 | ✅ 正常 | MI300X 原生支持，7000 系列可用 |
| float16 | ✅ 正常 | 所有 AMD GPU 原生支持 |
| torch.compile | ✅ 可用 | PyTorch 2.5+ |
| 多卡 DDP | ✅ 正常 | `torchrun` + `nccl` (ROCm 版 RCCL) |
| cuDNN | ⚠️ 不适用 | `trainer_utils.py` 自动跳过 `torch.backends.cudnn` 调用 |
| auto_config | ✅ 正常 | `hardware_probe()` 通过 `torch.version.hip` 识别 AMD GPU |

### 4.5 推荐 AMD GPU

| GPU | 显存 | 推荐用途 | 性能参考 |
|-----|------|---------|---------|
| RX 7900 XTX | 24GB | 全流程训练 + 推理 | ≈ RTX 4080 |
| RX 7900 XT | 20GB | 全流程训练 | ≈ RTX 4070 Ti |
| RX 7800 XT | 16GB | Pretrain + SFT | ≈ RTX 4070 |
| Instinct MI250X | 128GB | 大 batch / 长序列 | ≈ A100 |
| Instinct MI300X | 192GB | 极限实验 | ≈ H100 |

### 4.6 常见 AMD 问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `torch.cuda.is_available() = False` | ROCm 未正确安装或 PyTorch 不是 ROCm 版本 | `pip list | grep torch` 确认版本 |
| `Undefined symbol: rocblas_*` | ROCm 库版本不匹配 | 重装 PyTorch ROCm 版本 |
| `AttributeError: module 'torch.backends' has no attribute 'cudnn'` | AMD 没有 cuDNN | 已在代码中处理, 不应出现 |

## 五、CPU 部署

### 5.1 安装

```bash
pip install torch==2.6.0 --index-url https://download.pytorch.org/whl/cpu
```

### 5.2 使用

```bash
# 自动检测 — GPU 不可用时自动用 CPU
python trainer/train_full_sft.py

# 强制 CPU
python trainer/train_full_sft.py --device cpu
```

> CPU 训练约为 GPU 的 10-50 倍时间。仅适合小规模验证流程的正确性，完整训练强烈建议使用 GPU (NVIDIA 或 AMD)。

## 六、Docker 部署

### 6.1 镜像结构

```
pytorch/pytorch:2.6.0-cuda12.4-cudnn9-runtime  ← 基础镜像
  ├── base: Python + PyTorch + 系统工具
  ├── dev:  base + pip install -e ".[all]"     ← 开发/训练
  └── prod: base + pip install -e ".[serve]"    ← 生产服务
```

### 6.2 构建与运行

```bash
# 构建开发版
docker build -t minimind-lab:dev --target dev .

# 构建生产版 (更小)
docker build -t minimind-lab:prod --target prod .

# 运行 (挂载本地 out 目录)
docker run --gpus all -p 8000:8000 -v $(pwd)/out:/app/out minimind-lab:dev

# 直接启动 API 服务
docker run --gpus all -p 8000:8000 minimind-lab:prod minimind-serve

# 进入容器调试
docker run --gpus all -it minimind-lab:dev bash
```

### 6.3 Docker Compose (可选)

```yaml
# docker-compose.yml
services:
  minimind:
    build:
      context: .
      target: dev
    ports:
      - "8000:8000"
    volumes:
      - ./out:/app/out
      - ./dataset:/app/dataset
      - ./checkpoints:/app/checkpoints
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```

## 七、Makefile 操作手册

### 安装

```bash
make install        # pip install -e .
make install-train  # pip install -e ".[train]"
make install-all    # pip install -e ".[all]"
```

### 训练 (通过 ARGS 传参)

```bash
make pretrain       ARGS="--epochs 3 --batch_size 64"
make full-sft       ARGS="--epochs 5 --batch_size 32"
make lora           ARGS="--lora_name medical --epochs 10"
make distillation   ARGS="--alpha 0.5 --temperature 1.5"
make dpo            ARGS="--beta 0.15 --epochs 1"
make grpo           ARGS="--num_generations 6 --loss_type cispo"
make ppo            ARGS="--num_generations 6"
make agent          ARGS="--thinking_ratio 0.1"
```

### 推理

```bash
make eval           ARGS="--weight full_sft"
make serve          ARGS=""          # → http://localhost:8000
make chat           ARGS=""          # 交互对话
make auto-config    ARGS="--task full_sft --data_path dataset/sft.jsonl"
```

### 质量

```bash
make lint           # ruff check
make format         # ruff format
make test           # pytest tests/ -v
make clean          # 清除缓存和临时文件
```

### Docker

```bash
make docker-build   # docker build
make docker-run     # docker run
```

## 八、环境变量配置

复制 `.env.example` 为 `.env` 并根据需要修改：

```bash
cp .env.example .env
```

| 变量 | 说明 | 默认 |
|------|------|------|
| `MINIMIND_LOG_LEVEL` | 日志级别 | `INFO` |
| `MINIMIND_LOG_FILE` | 日志文件路径 | 空 (仅终端) |
| `MINIMIND_DATA_DIR` | 数据集目录 | `./dataset` |
| `MINIMIND_DEVICE` | 强制设备 | 空 (自动检测) |
| `HF_HOME` | HuggingFace 缓存 | `~/.cache/huggingface` |
| `WANDB_API_KEY` | Wandb API Key | 空 |

## 九、常见故障排查

### 网络问题

```bash
# 国内 pip 镜像加速
pip install -e . -i https://mirrors.aliyun.com/pypi/simple

# HuggingFace 镜像
export HF_ENDPOINT=https://hf-mirror.com

# Git 克隆加速
git clone https://gitee.com/mirrors/MiniMind-Lab  # 或 GitHub 镜像
```

### 显存不足

```bash
# 策略1: 减小 batch_size
--batch_size 8

# 策略2: 减小模型
--hidden_size 512 --num_hidden_layers 4

# 策略3: 减小序列长度
--max_seq_len 340

# 策略4: 启用梯度累积 (等效 batch = bs × accum)
--batch_size 4 --accumulation_steps 8  # 等效 batch=32

# 策略5: 使用 CPU offload (如果代码支持)
```

### 端口占用

```bash
# 启动 API 前检查
lsof -i :8000
# 或
netstat -an | grep 8000

# 修改端口
python scripts/serve_openai_api.py --port 8001
```

### PyTorch 版本检查

```bash
python -c "import torch; print(torch.__version__); print('CUDA:', torch.version.cuda); print('HIP:', torch.version.hip if hasattr(torch.version, 'hip') else 'N/A')"
```

### 找不到 Module

```bash
# 确保在项目根目录
cd MiniMind-Lab
pip install -e .  # 可编辑安装

# 或手动添加路径 (不推荐)
export PYTHONPATH="${PYTHONPATH}:$(pwd)"
```
