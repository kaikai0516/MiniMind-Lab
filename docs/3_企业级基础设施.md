## 二、YAML 配置系统

**文件**: `trainer/config.py` + **根目录 `config.yaml`**

### 配置优先级

```
config.yaml 默认值 → 用户自定义 YAML → CLI 参数
```

### 使用方式

```bash
# 方式1: 直接修改 config.yaml，然后照常运行
python trainer/train_full_sft.py

# 方式2: 临时 CLI 覆盖
python trainer/train_full_sft.py --epochs 10 --batch_size 64

# 方式3: 自定义 YAML + CLI 覆盖
python trainer/train_full_sft.py --config my_experiment.yaml --epochs 5
```

### Python API

```python
from trainer.config import load_config

config = load_config("full_sft", ["--epochs", "5"])
print(config.learning_rate)   # 1e-5
print(config.batch_size)      # 32
print(config.device)          # cuda:0 (自动检测)
```

## 三、Checkpoint 管理器

**文件**: `trainer/checkpoint.py`

`safetensors` 优先，自动轮转旧检查点。

```python
from trainer.checkpoint import LMCheckpointer

ckp = LMCheckpointer(
    save_dir="../checkpoints",
    max_keep=3,         # 最多保留 3 个
    fmt="safetensors"   # 或 "torch"
)

# 保存
ckp.save(model, optimizer, epoch=0, step=100, weight_name="full_sft")

# 加载
state = ckp.load("full_sft")           # 自动尝试 .safetensors → .pth
meta = ckp.load_metadata("full_sft")   # {epoch, step, config_hash, timestamp}

# 列出所有
for name, step, ts in ckp.list_checkpoints():
    print(name, step, ts)
```

| 特性 | 说明 |
|------|------|
| 格式 | safetensors (默认) / torch |
| 原子写入 | `.tmp` → `os.replace` |
| 元数据 | `_meta.json` (epoch, step, config_hash, timestamp) |
| 轮转 | `max_keep` 自动清理旧文件 |
