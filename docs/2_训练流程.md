## 一、训练全景图

```
[必须] 预训练     →  语言建模能力
  ↓
[必须] 全量 SFT   →  对话/指令跟随
  ↓
[可选] 蒸馏       →  小模型从大模型学  (需要教师模型)
[可选] LoRA       →  低资源领域适配    (医疗/法律/自我认知)
[可选] DPO        →  人类偏好对齐
[可选] GRPO       →  群组相对策略优化
[可选] PPO        →  近端策略优化
[可选] Agent RL   →  工具调用能力
```

**所有脚本在 `trainer/` 目录下执行**。输出权重保存在 `../out/`，checkpoint 保存在 `../checkpoints/`。

## 二、第1步: 预训练

### 原理

在大规模文本语料上做自回归因果语言建模 (Causal LM)：给定前文，预测下一个 token。目标是最小化交叉熵损失。预训练后的模型掌握了语言的统计规律——语法、常识、推理模式——但不具备对话能力。

### 完整命令

```bash
cd trainer

# 最小复现 (~2h on RTX 3090 24GB / RX 7900 XTX 24GB)
python train_pretrain.py \
    --data_path ../dataset/pretrain_t2t_mini.jsonl \
    --batch_size 32 \
    --learning_rate 5e-4 \
    --epochs 2 \
    --max_seq_len 768 \
    --dtype bfloat16 \
    --num_workers 8 \
    --log_interval 100 \
    --save_interval 1000

# 完整训练 (更大数据 + 更大 batch)
python train_pretrain.py \
    --data_path ../dataset/pretrain_t2t.jsonl \
    --hidden_size 768 \
    --num_hidden_layers 8 \
    --batch_size 64 \
    --learning_rate 5e-4 \
    --epochs 3 \
    --accumulation_steps 1 \
    --grad_clip 1.0
```

### 完整参数表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data_path` | `../dataset/pretrain_t2t_mini.jsonl` | 数据路径 (支持多文件) |
| `--hidden_size` | `768` | 隐藏层维度 (越大模型越强但越慢) |
| `--num_hidden_layers` | `8` | Transformer 层数 |
| `--use_moe` | `0` | 是否启用 MoE (0/1) |
| `--max_seq_len` | `768` | 训练最大序列长度 |
| `--batch_size` | `64` | 每卡 batch size |
| `--learning_rate` | `5e-4` | 初始学习率 |
| `--epochs` | `3` | 训练轮数 |
| `--accumulation_steps` | `1` | 梯度累积步数 (等效 batch = bs × accum) |
| `--grad_clip` | `1.0` | 梯度裁剪阈值 |
| `--dtype` | `bfloat16` | 混合精度: `bfloat16` / `float16` |

## 三、第2步: 全量 SFT 微调

### 原理

Supervised Fine-Tuning: 在对话数据上微调预训练模型。每条数据由 user-assistant 对话轮次组成，模型只对 assistant 回复计算 loss (user 部分被 mask 掉)。SFT 后模型具备对话和指令跟随能力。

### 完整命令

```bash
python train_full_sft.py \
    --data_path ../dataset/sft_t2t_mini.jsonl \
    --from_weight pretrain \
    --save_weight full_sft \
    --batch_size 32 \
    --learning_rate 1e-5 \
    --epochs 3 \
    --max_seq_len 768 \
    --dtype bfloat16
```

### 完整参数表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data_path` | `../dataset/sft_t2t_mini.jsonl` | 数据路径 |
| `--from_weight` | `pretrain` | 从哪个预训练权重加载 |
| `--save_weight` | `full_sft` | 输出权重名 |
| `--batch_size` | `32` | batch size |
| `--learning_rate` | `1e-5` | SFT 学习率 (比预训练低) |
| `--epochs` | `3` | 训练轮数 |
| `--max_seq_len` | `768` | 最大序列长度 |
| `--accumulation_steps` | `1` | 梯度累积 |
| `--grad_clip` | `1.0` | 梯度裁剪 |
| `--dtype` | `bfloat16` | 精度 |
| `--num_workers` | `8` | 数据加载线程 |

## 四、第3步: 知识蒸馏

### 原理

用一个更大/更强的教师模型 (Teacher) 指导小模型 (Student) 学习。核心思想：
- **硬标签 (CE Loss)**: Student 正常做 next-token prediction
- **软标签 (KL Loss)**: Student 的输出概率分布对齐 Teacher 的输出概率分布
- **总 Loss** = α × KL_Loss + (1-α) × CE_Loss

Temperature 温度参数控制概率分布"软"的程度：温度越高，分布越平滑，次要 token 获得更多权重。

### 完整命令

```bash
python train_distillation.py \
    --data_path ../dataset/sft_t2t_mini.jsonl \
    --student_hidden_size 768 \
    --student_num_layers 8 \
    --teacher_hidden_size 768 \
    --teacher_num_layers 8 \
    --teacher_use_moe 1 \
    --student_use_moe 0 \
    --from_student_weight full_sft \
    --from_teacher_weight full_sft \
    --alpha 0.5 \
    --temperature 1.5 \
    --save_weight full_dist \
    --batch_size 32 \
    --learning_rate 5e-6 \
    --epochs 6
```

### 蒸馏专属参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--student_hidden_size` | `768` | 学生模型隐藏维度 |
| `--student_num_layers` | `8` | 学生模型层数 |
| `--teacher_hidden_size` | `768` | 教师模型隐藏维度 |
| `--teacher_num_layers` | `8` | 教师模型层数 |
| `--student_use_moe` | `False` | 学生是否 MoE |
| `--teacher_use_moe` | `True` | 教师是否 MoE |
| `--from_student_weight` | `full_sft` | 学生起始权重 |
| `--from_teacher_weight` | `full_sft` | 教师起始权重 |
| `--alpha` | `0.5` | 蒸馏损失权重 (0=纯CE, 1=纯KL) |
| `--temperature` | `1.5` | 蒸馏温度 (越高分布越软) |

## 五、第4步: LoRA 微调

### 原理

Low-Rank Adaptation: 冻结预训练模型所有参数，只在每个线性层注入低秩可训练矩阵 (A·B)。只需要训练极少参数 (通常 < 1%) 就能在特定领域达到接近全量微调的效果。

### 完整命令

```bash
python train_lora.py \
    --data_path ../dataset/lora_medical.jsonl \
    --from_weight full_sft \
    --lora_name lora_medical \
    --save_weight lora_medical \
    --batch_size 32 \
    --learning_rate 1e-4 \
    --epochs 10 \
    --max_seq_len 340
```

### LoRA 专属参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--lora_name` | `lora_medical` | LoRA 权重名称 |
| `--max_seq_len` | `340` | LoRA 训练序列较短 (显存优化) |

## 六、第5步: DPO 偏好对齐

### 原理

Direct Preference Optimization: 不需要训练奖励模型，直接从偏好对 (chosen/rejected) 中学习。核心是用一个巧妙的数学推导，将 RLHF 中的 Reward Model → RL 两步融合为一步。

DPO Loss: L = -log σ(β · (log π(chosen)/π_ref(chosen) - log π(rejected)/π_ref(rejected)))

其中 β 控制偏离参考模型的程度，过大会过于保守，过小会退化。

### 完整命令

```bash
python train_dpo.py \
    --data_path ../dataset/dpo.jsonl \
    --from_weight full_sft \
    --save_weight dpo \
    --batch_size 4 \
    --learning_rate 4e-8 \
    --beta 0.15 \
    --epochs 1 \
    --max_seq_len 1024
```

### DPO 专属参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--beta` | `0.15` | KL 散度惩罚系数 |
| `--batch_size` | `4` | DPO 显存需求高，batch 较小 |
| `--learning_rate` | `4e-8` | DPO 学习率极低 |

## 七、第6步: GRPO 强化学习

### 原理

Group Relative Policy Optimization (GRPO): 群组相对策略优化。对每个 prompt 生成 N 个 response，用奖励模型打分，然后根据组内相对排名计算优势函数来更新策略。

**关键步骤**:
1. Rollout: 模型对每个 prompt 生成 num_generations 个 response
2. Scoring: 奖励模型给每个 response 打分
3. Advantage: 组内归一化 — advantage = (reward - mean) / std
4. Policy Loss: L = -E[min(ratio × advantage, clip(ratio, 1-ε, 1+ε) × advantage)]

### GRPO vs CISPO

| | GRPO | CISPO |
|------|------|-------|
| Clip 方式 | 固定 ε | ε_low 和 ε_high 不对称 |
| 适用场景 | 通用 | 需要更保守更新的场景 |
| `--loss_type` | `grpo` | `cispo` (默认) |

### 完整命令

```bash
python train_grpo.py \
    --data_path ../dataset/rlaif.jsonl \
    --from_weight full_sft \
    --save_weight grpo \
    --batch_size 2 \
    --learning_rate 3e-7 \
    --num_generations 6 \
    --max_gen_len 1024 \
    --beta 0.1 \
    --loss_type cispo \
    --epsilon 0.2 \
    --epsilon_high 5.0 \
    --thinking_ratio 0.9
```

### GRPO 专属参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--num_generations` | `6` | 每个 prompt 生成 response 数量 |
| `--max_gen_len` | `1024` | 单次生成最大 token 数 |
| `--beta` | `0.1` | KL 惩罚系数 |
| `--loss_type` | `cispo` | `cispo` (推荐) / `grpo` |
| `--epsilon` | `0.2` | GRPO clip epsilon |
| `--epsilon_high` | `5.0` | CISPO 上界 epsilon |
| `--thinking_ratio` | `0.9` | 随机启用 `<think>` 标签概率 |
| `--rollout_engine` | `torch` | 生成引擎: `torch` / `sglang` |

## 八、第7步: PPO 强化学习

### 原理

Proximal Policy Optimization: 经典的 clipped surrogate objective。与 GRPO 的区别在于：
- PPO 使用 GAE (Generalized Advantage Estimation) 计算优势
- PPO 只生成 1 个 response per prompt (GRPO 生成 N 个)
- PPO 需要 Critic (Value) 模型来估计状态价值
- 因此 PPO 有 actor_loss + critic_loss

### 完整命令

```bash
python train_ppo.py \
    --data_path ../dataset/rlaif.jsonl \
    --from_weight full_sft \
    --save_weight ppo \
    --batch_size 2 \
    --learning_rate 3e-7 \
    --num_generations 6 \
    --max_gen_len 1024 \
    --beta 0.1 \
    --loss_type cispo
```

## 九、第8步: Agent RL

### 原理

训练模型使用外部工具 (如计算器、搜索引擎)。数据中包含 `<tool_call>` 标签，模型学会在需要时生成工具调用，接收工具返回结果后继续推理。

### 完整命令

```bash
python train_agent.py \
    --data_path ../dataset/agent_rl.jsonl \
    --from_weight full_sft \
    --save_weight agent \
    --batch_size 2 \
    --learning_rate 3e-7 \
    --num_generations 4 \
    --max_gen_len 768 \
    --max_total_len 2500 \
    --thinking_ratio 0.1 \
    --epochs 1
```

### Agent 专属参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--max_gen_len` | `768` | 单次生成最大长度 |
| `--max_total_len` | `2500` | 整个交互最大总长度 |
| `--num_generations` | `4` | Agent 场景通常更少 generation |
| `--thinking_ratio` | `0.1` | Agent 场景低思考比例 |

## 十、多卡训练 (DDP)

MiniMind-Lab 的 DDP 同时支持 NVIDIA 和 AMD GPU：

```bash
# 单机 N 卡
torchrun --nproc_per_node N train_full_sft.py \
    --batch_size 32 --epochs 3

# 多机多卡 (2 机器 × 4 卡)
# 主节点 (192.168.1.100):
torchrun --nproc_per_node 4 --nnodes 2 \
    --node_rank 0 --master_addr 192.168.1.100 --master_port 29500 \
    train_pretrain.py --batch_size 64

# 从节点 (192.168.1.101):
torchrun --nproc_per_node 4 --nnodes 2 \
    --node_rank 1 --master_addr 192.168.1.100 --master_port 29500 \
    train_pretrain.py --batch_size 64
```

### DDP 关键细节

- `wrap_model()` 自动处理 `compile → DDP → rollout_engine` 的正确顺序
- `SkipBatchSampler` 保证续训时跳过已完成的 batch
- `DistributedSampler` 在每个 epoch 开头需要 `set_epoch(epoch)` 以保证 shuffle 种子不同

## 十三、RL 算法速查

| 算法 | 核心思想 | 需要 Critic | 每 Prompt 生成数 | 论文 |
|------|---------|------------|----------------|------|
| DPO | 直接从偏好对学习，无需奖励模型 | ❌ | 0 (离线) | Rafailov et al. 2023 |
| GRPO | 组内相对排名 + clip | ❌ | 6+ | — |
| CISPO | 不对称 clip | ❌ | 6+ | — |
| PPO | GAE + clip + Critic | ✅ | 1+ | Schulman et al. 2017 |
| Agent RL | 工具调用 + RL | ❌ | 4 | — |

### RL 通用参数速查

| 参数 | 适用 | 默认 | 作用 |
|------|------|------|------|
| `--beta` | DPO/GRPO/PPO/Agent | 0.1-0.15 | KL 惩罚 — 越大越保守 |
| `--num_generations` | GRPO/PPO/Agent | 4-6 | 每 prompt 采样数 |
| `--max_gen_len` | GRPO/PPO/Agent | 768-1024 | 单 response 最大长度 |
| `--loss_type` | GRPO/PPO | `cispo` | 损失函数变体 |
| `--epsilon` | GRPO/PPO | 0.2 | clip 阈值 |
| `--thinking_ratio` | GRPO/Agent | 0.1-0.9 | 思考标签概率 |
