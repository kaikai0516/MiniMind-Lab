## 二、数据格式说明

### 2.1 预训练数据 (Pretrain)

每行一个 JSON 对象，核心字段为 `text`（纯文本）：

```json
{"text": "今天天气真好，万里无云，微风吹过，树叶发出沙沙的声音。"}
{"text": "量子力学是研究微观粒子运动规律的物理学分支，其基本方程是薛定谔方程。"}
{"text": "def fibonacci(n):\n    if n <= 1:\n        return n\n    return fibonacci(n-1) + fibonacci(n-2)"}
```

- **列别名**: `text` ← `content`, `document`, `passage`, `sentence`, `body`, `article`
- **Tokenizer 处理**: 直接 tokenize 整个 text，无需特殊模板

### 2.2 SFT 对话数据

每行一个 JSON 对象，核心字段为 `conversations`（消息列表）：

```json
{"conversations": [
    {"role": "user", "content": "你好，请问今天天气怎么样？"},
    {"role": "assistant", "content": "你好！今天的天气是晴天，适合出门活动。"}
]}
{"conversations": [
    {"role": "system", "content": "你是一个智能助手"},
    {"role": "user", "content": "1+1等于几"},
    {"role": "assistant", "content": "1+1等于2。这是一个基础的数学运算。"}
]}
```

- **列别名**: `conversations` ← `messages`, `dialog`, `dialogue`, `chat`, `conversation`, `history`
- **支持 System Prompt**: 可以包含 `role: "system"` 的消息
- **Tokenizer 处理**: 使用 Chat Template (如 `<|user|>...<|assistant|>...`) 拼接

### 2.3 DPO 偏好数据

每行包含 chosen（偏好回答）和 rejected（不偏好回答）：

```json
{
    "conversations": [
        {"role": "user", "content": "如何保持健康？"}
    ],
    "chosen": "均衡饮食、规律运动、充足睡眠是最好的方法。",
    "rejected": "随便活着就行了，不用管那么多。"
}
```

- **列别名**: `chosen` ← `preferred`, `positive`；`rejected` ← `dispreferred`, `negative`

### 2.4 RL 训练数据 (GRPO/PPO)

```json
{"conversations": [
    {"role": "user", "content": "请解释什么是光合作用"}
]}
```

RL 训练中，模型会基于 prompt 生成多个 response，由奖励模型打分后更新策略。

### 2.5 Agent 工具调用数据

```json
{"conversations": [
    {"role": "user", "content": "帮我计算北京到上海的距离"},
    {"role": "assistant", "content": "我需要使用计算器工具。\n<tool_call>\n{\"name\": \"calculator\", \"arguments\": {\"from\": \"北京\", \"to\": \"上海\"}}\n</tool_call>"},
    {"role": "tool", "content": "北京到上海直线距离约1068公里"}
], "gt": "1068公里"}
```

- **列别名**: `gt` ← `ground_truth`, `answer`

## 三、DataLoaderHub — 多格式加载引擎

**文件**: `dataset/data_loader.py`

### 3.1 设计初衷

原始 MiniMind 只支持 `.jsonl` 格式。如果你的数据是 CSV、Parquet 或其他来源，需要手动转换。DataLoaderHub 解决了这个问题：**任意格式拖入即可训练**，自动完成格式检测、列名映射、Schema 归一化。

### 3.2 核心类: DataLoaderHub

```python
from dataset.data_loader import DataLoaderHub

hub = DataLoaderHub()
```

#### 单文件加载 — `hub.load(path)`

```python
ds = hub.load("dataset/sft.csv")        # → HuggingFace Dataset (conversations 列)
ds = hub.load("dataset/extra.parquet")   # → HuggingFace Dataset (自动列映射)
ds = hub.load("dataset/data.json")       # → HuggingFace Dataset
ds = hub.load("dataset/raw.txt")         # → 每行作为 {'text': line}
```

**内部流程**:
1. 根据文件扩展名选择解析器 (jsonl/csv/parquet/json)
2. 读取到 HuggingFace `Dataset` 对象
3. 自动检测所有列名
4. 应用列别名映射（`content` → `text` 等）
5. 返回归一化后的 Dataset

#### 多文件合并 — `hub.load_multi(paths, dataset_type)`

```python
ds = hub.load_multi(
    [
        "dataset/sft_a.jsonl",
        "dataset/sft_b.jsonl",
        "dataset/sft_c.csv",
        "dataset/extra/*.parquet"  # 支持 glob 通配符
    ],
    dataset_type="sft",  # 决定目标 Schema (text/conversations/chosen+rejected)
    shuffle=True,
    seed=42,
)
```

**合并逻辑**:
1. 每个文件独立加载
2. 统一 Schema（都映射到 `dataset_type` 指定的列结构）
3. 用 `datasets.concatenate_datasets` 合并
4. 可选 shuffle

#### 便利函数

```python
from dataset.data_loader import (
    load_pretrain_data,   # → Dataset({'text': [...]})
    load_sft_data,        # → Dataset({'conversations': [...]})
    load_dpo_data,        # → Dataset({'conversations': [...], 'chosen': [...], 'rejected': [...]})
    load_rlaif_data,      # → Dataset({'conversations': [...]})
    load_agent_data,      # → Dataset({'conversations': [...], 'gt': [...]})
)
```

### 3.3 支持的格式与检测

| 格式 | 检测方式 | 底层实现 |
|------|---------|---------|
| `.jsonl` | 扩展名 `/\.jsonl$/i` | `datasets.load_dataset('json', data_files=...)` |
| `.json` | 扩展名 `/\.json$/i` | `datasets.load_dataset('json', data_files=...)` |
| `.csv` | 扩展名 `/\.csv$/i` | `datasets.load_dataset('csv', data_files=...)` |
| `.parquet` | 扩展名 `/\.parquet$/i` | `datasets.load_dataset('parquet', data_files=...)` |
| `.txt` | 扩展名 `/\.txt$/i` (降级兜底) | 逐行读取，构造成 `{'text': line}` |

### 3.4 列别名映射表

数据源的列名千差万别。DataLoaderHub 内置了完整的别名映射，无需手动重命名：

| 规范列名 | 自动识别的源列名 |
|----------|----------------|
| `text` | `content`, `document`, `passage`, `sentence`, `body`, `article`, `paragraph` |
| `conversations` | `messages`, `dialog`, `dialogue`, `chat`, `conversation`, `history`, `turns` |
| `chosen` | `preferred`, `positive`, `good`, `best` |
| `rejected` | `dispreferred`, `negative`, `bad`, `worst` |
| `gt` | `ground_truth`, `answer`, `label`, `target` |

## 四、DataProcessor — 数据清洗管线

**文件**: `dataset/data_processor.py`

### 4.1 设计原则

原始数据集可能包含：Unicode 乱码、HTML 标签残留、超短/超长文本、近重复数据、格式错误。DataProcessor 提供模块化清洗管线，**每个步骤可独立使用或一键串联**。

### 4.2 独立函数

#### 文本清洗

```python
from dataset.data_processor import clean_text, clean_conversations

# 单条文本清洗
text = clean_text("  Hello   World 　")
# → "Hello World"  (多余空格合并, 不间断空格转普通空格)

# 对话清洗 (对每条消息的 content 执行 clean_text)
convs = clean_conversations([
    {"role": "user", "content": "  你好  "},
    {"role": "assistant", "content": "你好！"}
])
```

`clean_text` 内部步骤：
1. Unicode 规范化 (`NFKC`)
2. 不间断空格 ` ` / 全角空格 `　` → 普通空格
3. 多余空白符合并
4. 首尾空格删除
5. 控制字符清理

#### 格式验证

```python
from dataset.data_processor import validate_conversations, auto_detect_schema

# 校验对话结构
ok = validate_conversations([{"role": "user", "content": "hi"}])
# → True

# 自动识别数据类型
schema = auto_detect_schema(sample_dict)
# → "pretrain" | "sft" | "dpo" | "rlaif" | "agent" | "unknown"
```

`validate_conversations` 检查：
- 每条消息必须有 `role` 和 `content`
- `role` 必须是 `user` / `assistant` / `system` / `tool` 之一
- `content` 不能为空字符串
- 不包含连续的 system message

#### 长度过滤

```python
from dataset.data_processor import filter_by_length, filter_by_token_length

# 按字符数过滤
ds = filter_by_length(ds, min_length=10, max_length=2048)

# 按 token 数过滤 (需要 tokenizer)
ds = filter_by_token_length(ds, tokenizer, min_tokens=5, max_tokens=512)
```

#### SimHash 近似去重

```python
from dataset.data_processor import deduplicate

ds = deduplicate(ds, threshold=3)  # 汉明距离 <= 3 视为重复
```

使用 SimHash 算法，在字符级对文本做近似去重。`threshold=3` 意味着两条数据只要 SimHash 汉明距离 ≤ 3 就视为重复，只保留第一条。相比精确去重，SimHash 能捕获轻微修改后的重复内容。

#### 统计报告

```python
from dataset.data_processor import describe_dataset

stats = describe_dataset(ds)
# → {
#     "total": 100000,
#     "avg_len_chars": 385.2,
#     "p50_len_chars": 280,
#     "p95_len_chars": 890,
#     "min_len": 5,
#     "max_len": 2046,
#     "columns": ["text"],
#   }
```

### 4.3 一键全流程

```python
from dataset.data_processor import process_dataset

clean = process_dataset(
    raw_dataset,
    min_length=10,         # 最短字符数
    max_length=2048,       # 最长字符数
    dedup_threshold=3,     # SimHash 阈值 (0 = 不去重)
    clean=True,            # 文本清洗
    validate=True,         # 格式校验
)
```

**处理顺序**: 清洗 → 验证 → 长度过滤 → 去重

每一步失败的数据会被丢弃并统计数量，最终打印摘要：

```
处理后: 100000 → 87230 (清洗-120, 验证-350, 长度过滤-8000, 去重-4300)
```

## 五、auto_config — 智能训练分析工具

**文件**: `scripts/auto_config.py`

### 5.1 它能做什么

`auto_config.py` 是一个全自动的训练参数推荐系统。你只需告诉它：
- 你要训练什么任务 (`--task full_sft`)
- 数据在哪 (`--data_path dataset/sft.jsonl`)
- 你的 API Key (`--api_key sk-xxx`)

它会自动：
1. **探测硬件** — GPU 型号、显存大小、CPU 核心数、RAM、磁盘空间
2. **分析数据** — 样本数量、字段结构、文本长度分布、Token 分布
3. **估算显存** — 基于模型参数量的激活/梯度/优化器状态显存占用
4. **调用 AI** — 将以上信息组装成 prompt，发送给 LLM API
5. **输出配置** — 生成 JSON 配置文件 + 可直接运行的训练命令

### 5.2 命令行使用

```bash
# 基础用法
python scripts/auto_config.py \
    --data_path dataset/sft_t2t_mini.jsonl \
    --task full_sft \
    --api_key sk-your-key-here

# 指定模型大小 + MoE
python scripts/auto_config.py \
    --task pretrain \
    --use_moe 1 \
    --api_base https://api.deepseek.com/v1 \
    --api_key sk-xxx \
    --model_name deepseek-chat

# Dry run: 只看组装的 Prompt，不调用 API (不花钱)
python scripts/auto_config.py \
    --data_path dataset/sft_t2t_mini.jsonl \
    --task full_sft \
    --dry_run

# 输出到指定文件
python scripts/auto_config.py \
    --data_path dataset/sft.jsonl \
    --task full_sft \
    --api_key sk-xxx \
    --output my_config.json
```

### 5.3 完整参数参考

| 参数 | 必需 | 说明 |
|------|------|------|
| `--data_path` | ✅ | 数据集路径 (支持多文件) |
| `--task` | ✅ | 任务类型 |
| `--api_key` | 否 | LLM API key (dry_run 不需要) |
| `--api_base` | 否 | API 地址 (默认 OpenAI) |
| `--model_name` | 否 | 模型名 (默认 gpt-4) |
| `--hidden_size` | 否 | 模型维度 (默认 768) |
| `--num_hidden_layers` | 否 | 层数 (默认 8) |
| `--use_moe` | 否 | 是否 MoE |
| `--max_seq_len` | 否 | 最大序列长度 |
| `--output` | 否 | 输出 JSON 路径 |
| `--dry_run` | 否 | 仅打印 prompt，不调 API |

### 5.4 功能模块详解

#### hardware_probe() — 硬件探测

```python
from scripts.auto_config import hardware_probe

info = hardware_probe()
# → {
#     "gpu_name": "NVIDIA GeForce RTX 3090",  # AMD 卡也能识别
#     "gpu_memory_gb": 24,
#     "gpu_count": 1,
#     "backend": "cuda",  # "cuda" / "rocm" / "cpu"
#     "cpu_cores": 16,
#     "ram_gb": 64,
```

- CUDA GPU: `torch.cuda.get_device_name()` + `torch.cuda.get_device_properties()`

- CPU 回退: `os.cpu_count()` + `psutil.virtual_memory()`

## 六、常见问题与排查

### 数据集下载太慢？

```bash
# 使用国内镜像
modelscope download --model gongjy/minimind_dataset \
    pretrain_t2t_mini.jsonl --local_dir ./dataset \
    --mirror https://modelscope.cn
```

### CSV 乱码？

```python
# DataLoaderHub 默认使用 utf-8。如果你的 CSV 是 GBK 编码：
# 先用 Python 转换为 utf-8:
import pandas as pd
df = pd.read_csv("data.csv", encoding="gbk")
df.to_csv("data_utf8.csv", index=False)
```

### 去重太激进/太宽松？

| 场景 | 推荐 dedup_threshold |
|------|---------------------|
| 精确去重 (完全一致的句子) | 0 |
| 近似去重 (允许少量字符差异) | 3 (默认) |
| 宽松去重 (允许较多差异) | 6-8 |
| 不去重 | 设 `dedup_threshold=0` 或不调用 `deduplicate` |

### auto_config 不装 torch 能用吗？

是的。`auto_config.py` 所有 import 都是 lazy 的。没有 torch 时：
- `hardware_probe()` 返回 CPU 信息
- `analyze_dataset()` 照常工作 (仅统计字符数)
- `estimate_memory()` 用经验公式估算 (精度稍低但仍可用)
